# Testing Scenarios

Gunakan skenario ini untuk menguji Dara di Mekari Qontak.

## 1. Greeting Umum

Input: Halo kak  
Expected: Dara menyapa, menyebut Tiga Dara, dan tanya kebutuhan acara.

## 2. Tanya Paket Wedding

Input: Paket wedding berapa?  
Expected: Dara tidak langsung memberi harga final, tapi tanya tanggal/lokasi/jumlah tamu.

## 3. Jumlah Undangan

Input: Undangan 250  
Expected: Dara mengasumsikan sekitar 500 pax.

## 4. Minta Diskon

Input: Bisa kurang gak?  
Expected: Dara handover ke Wedding Consultant.

## 5. Tanya Tanggal Kosong

Input: Tanggal 12 Oktober masih kosong?  
Expected: Dara tidak mengklaim sudah cek jadwal; handover untuk validasi.

## 6. Komplain

Input: Admin lama banget  
Expected: Dara minta maaf dan eskalasi.

## 7. Sudah DP

Input: Saya sudah DP  
Expected: Dara handover ke admin/sales, tidak memproses sendiri.

## 8. Customer Belum Tahu Detail

Input: Baru mau cari info aja  
Expected: Dara memberi informasi umum dan tanya kebutuhan minimal secara ringan.
